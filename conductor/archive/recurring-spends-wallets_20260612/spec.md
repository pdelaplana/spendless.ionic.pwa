# Specification: Select Wallet for Recurring Spends on Period Creation

**Track ID:** recurring-spends-wallets_20260612
**Type:** Feature
**Created:** 2026-06-12
**Status:** Draft

## Summary

During new period creation, ensure that wallets are set up first, then present the user with the recurring spends/expenses configuration screen where they can explicitly select which wallet each recurring spend should be assigned to.

## Context

Spendless is a mindful spending tracker PWA that helps users understand their spending through emotional awareness. It focuses on the "why" of spending, using six mindful categories, wallet management, and flexible spending periods. Proper wallet assignment is critical to ensure that expenses are tracked against the correct financial accounts and that period limits/balances reflect reality.

## User Story

As a mindful spender setting up a new spending period, I want to choose which wallet each of my recurring spends is assigned to, so that my recurring expenses are correctly attributed and do not get assigned to the wrong wallets.

## Acceptance Criteria

- [ ] Wallets are created/loaded and verified before the recurring spends selection step.
- [ ] The recurring spends setup screen in the Period Creation Modal displays a wallet selector (e.g., dropdown or selection list) for each recurring expense.
- [ ] The user's selection of wallet is persisted when the recurring spends are cloned/created for the new period.
- [ ] Selecting a wallet updates the local state and correctly updates Firestore upon period modal completion.
